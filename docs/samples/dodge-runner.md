# Dodge Runner — はじめての Three.js ゲーム

奥から流れてくる赤いキューブを、左右に動かす青い球体で避け続けるシンプルなゲームです。

## 操作方法

| キー | 動作 |
| --- | --- |
| `←` / `A` | 左に移動 |
| `→` / `D` | 右に移動 |
| `R` | リスタート |

スコアは「生き延びた秒数」、進めば進むほど障害物の速度と出現頻度が上がっていきます。

## 学習ポイント

- **複数モジュールの組み合わせ**で 1 つのサンプルを作る方法
- **delta-time** に基づくフレームレート非依存の移動 / 生成ロジック
- **オブジェクトプール**で Mesh を使い回し、GC 負荷を抑える
- **状態機械**（`playing` / `gameover`）でゲームのフェーズを安全に切り替える
- **DOM オーバーレイ**による軽量な HUD 実装

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `index.ts` | Sample 契約のエントリ。各モジュールを配線するだけ |
| `scene.ts` | Scene / Camera / ライト / 床 / 走路の白線 |
| `player.ts` | プレイヤー（青い球体） |
| `obstacles.ts` | 赤いキューブの生成・移動・当たり判定（プール使用） |
| `input.ts` | キーボード入力を「状態」として保持 |
| `hud.ts` | スコア表示と Game Over オーバーレイ（DOM） |
| `game-state.ts` | スコアと状態機械 |

## 改造のヒント

ここまで読んだら、以下のような小さな改造に挑戦してみてください。

- `obstacles.ts` の `BASE_FORWARD_SPEED` を変えて難易度を調整
- `player.ts` の `MOVE_SPEED` を変えて、機敏な操作感にする
- `scene.ts` の `Fog` の色を背景と変えて、不思議な雰囲気にする
- `game-state.ts` の `computeSpeedMultiplier` のカーブを変えて難易度カーブを調整
- 障害物の色やサイズをランダムにして、見た目を多様にする

慣れてきたら、新しいゲームを `src/games/<your-game>/` 以下に同じパターンで作ってみましょう。
